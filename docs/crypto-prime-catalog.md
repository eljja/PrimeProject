# 암호 소수 카탈로그와 감사 체크리스트

작성일: 2026-05-17

## 목적

이 문서는 `prime_audit` 도구가 검사할 표준 소수, 생성기 편향, 약한 키 패턴을 정리한다. 범위는 방어적 키 품질 감사와 자체 실험 데이터 검증으로 제한한다.

## RSA 비밀 소수

RSA에서 공개되는 값은 보통 `n = p * q`이고, `p`, `q`는 비밀이다. 정상 구현에서는 `p`, `q`가 충분한 엔트로피로 독립 생성되어야 하며 공개키만으로 예측 가능하면 안 된다.

감사 항목:

- 같은 키셋 안에서 공유 소수 인자가 있는지 batch GCD로 확인한다.
- `n`이 완전제곱에 비정상적으로 가까운지 확인한다.
- bit length가 기대 범위인지 확인한다.
- `n mod small primes` 지문이 알려진 취약 생성기나 특정 생성기 모델과 지나치게 잘 맞는지 확인한다.
- 같은 발급자, 제조사, 생성 시점별로 충돌과 이상치를 묶어서 본다.

주요 취약 패턴:

- 저엔트로피 또는 예측 가능한 RNG.
- 동일 seed 또는 부팅 직후 seed 재사용.
- 서로 다른 키가 같은 `p` 또는 `q`를 공유.
- `p`, `q`가 너무 가까움.
- 특정 모듈러 잔여류에 갇히는 구조적 생성기 편향.

## 유한체 DH 공개 소수

FFDHE/MODP 그룹의 소수는 공개 도메인 파라미터다. 예측 대상이 아니라 검증 대상이다.

감사 항목:

- 표준 그룹 상수인지 확인한다.
- safe prime 조건 `p = 2q + 1`, `q` prime을 확인한다.
- 비표준 파라미터일 경우 bit length, subgroup 구조, generator 검증 여부를 별도 검토한다.

대표 표준:

- RFC 7919 FFDHE.
- RFC 3526 MODP.

## ECC prime field 소수

ECC field prime 역시 보통 공개 상수다. 소수 하나만으로 곡선 안전성이 결정되지 않으므로 곡선 차수, cofactor, base point, twist security, 구현 안정성을 함께 검토해야 한다.

도구의 초기 인식 대상:

- Curve25519: `2^255 - 19`
- Curve448: `2^448 - 2^224 - 1`
- NIST P-256 field prime: `2^256 - 2^224 + 2^192 + 2^96 - 1`
- secp256k1 field prime: `2^256 - 2^32 - 977`

## 점수 체계

`prime_audit`는 키를 "깬다"는 결론을 직접 내리지 않고, 감사 우선순위를 정하기 위한 점수를 제공한다.

- `shared_factor`: 같은 데이터셋 안에서 공유 인자가 발견되면 위험도 `critical`.
- `near_square`: Fermat 방식으로 빠르게 인수분해될 만큼 완전제곱에 가까우면 위험도 `high`.
- `weak_residue_fingerprint`: 잔여류 지문이 설정한 약한 모델과 일치하면 위험도 `medium` 이상.
- `standard_parameter`: 표준 DH/ECC 상수와 일치하면 정보성 판정.
- `unknown_public_prime`: 공개 도메인 소수가 표준 목록에 없으면 수동 검토 권고.

## 데이터 입력 원칙

초기 구현은 공개 값만 받는다.

- RSA: 공개 모듈러스 `n`, 선택적으로 `e`, 출처 메타데이터.
- DH/ECC: 공개 소수 `p`, 선택적으로 이름과 출처.
- 실제 운영 키를 분석할 때는 소유 권한이 있는 키셋만 사용한다.
